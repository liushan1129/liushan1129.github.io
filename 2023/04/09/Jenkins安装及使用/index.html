Jenkins安装及使用

John Doe
2023-04-09

安装JDK

Jenkins需要依赖JDK，所以先安装JDK1.8
yum install java-1.8.0-openjdk* -y

arch 安装使用 sudo pacman -S java-1.8.0-openjdk* -y

安装目录为：/usr/lib/jvm

获取jenkins安装包

因为jenkins 最新版本仅支持jdk11, 所以使用2.332.1版本
下载页面：https://mirrors.jenkins.io/war-stable/2.332.1/
下载jenkins.war
或
找到合适的下载地址
下载的安装文件：jenkins-2.190.3-1.1.noarch.rpm

把安装包上传到192.168.64.154服务器，进行安装

方式一、jenkins.war安装方式：
java -jar jenkins.war

方式二、rpm安装方式
rpm -ivh jenkins-2.190.3-1.1.noarch.rpm

**注意：如果使用官网的安装方法需要jdk11，jdk8会启动报错
*

war安装直接访问127.0.0.1:8080 进行安装，插件直接装上就行，没有很慢
【目前没有找到修改配置文件的地方！！！！】

以下操作仅使用rpm安装时操作

修改Jenkins配置

sudo vi /etc/syscofig/jenkins

修改内容如下：
JENKINS_USER=”root”
JENKINS_PORT=”8888”

启动Jenkins

systemctl start jenkins

防火墙添加指定端口

firewall-cmd –zone=public –add-port=8888/tcp –permanent
firewall-cmd –reload

6）打开浏览器访问
http://192.168.64.154:8888

安装时插件选择无，此时插件下载很慢后续在安装插件

安装插件

汉化后 系统管理 —> 插件管理 —> 可选插件
搜索自己要安装的插件 选择–install
重启jenkins
domain:port/restart

插件生效

问题二、 jenkins登录后空白页

  1. 访问安装了jenkins的主机ip
    http://192.168.64.154:8888/pluginManager/advanced
    在本机可以直接访问
    http://127.0.0.1:8888/pluginManager/advanced

将最下面 Update Site中 https 改为http

重新启动 jenkins
http://127.0.0.1:8888/restart

jenkins卸载

rpm卸载
rpm -e jenkins

检查是否卸载成功
rpm -ql jenkins

彻底删除残留文件
find / -iname jenkins | xargs -n 1000 rm -rf

手动删除没卸干净的文件
sudo rm -rf filename

扩展：关于防火墙其他操作命令

查看防火墙状态：
systemctl status firewalld

开启防火墙
systemctl start firewalld

关闭防火墙
systemctl stop firewalld

重新加载防火墙
firewall-cmd –reload

查看防火墙已开放端口
firewall-cmd –list-port

添加指定端口
firewall-cmd –zone=public –add-port=这里是需要开启的端口号/tcp –permanent
