static

static只能修饰在类里定义的成员部分，包括成员变量、方法、内部类（枚举或接口）、初始化块。如过没有使用static修饰类里的这些成员，这些成员属于该类的实例（随对象的初始化而初始化，即new 对象时）；如果使用static修饰，这些成员就属于类本身（随类的初始化而初始化）。

static只能修饰类里的成员，不能修饰外部类，不能修饰局部变量、局部内部类。

static修饰的成员变量只会初始化一次， 没有用static修饰的成员变量在每次创建对象时都会初始化。

super 和 this

super调用 用于显示调用父类构造器, this调用 用于显示调用本类中另一个重载的构造器。
super调用和this调用都只能在构造器中使用，而且必须作为构造器的第一行代码，因此构造器中的super调用和this调用最多只能使用其中一个，而且最多只能调用一次。

在程序中创建java对象，系统总是先调用顶层父类的初始化操作。包括初始化块和构造器，然后依次向下调用所有父类的初始化操作，最终执行本类的初始化操作返回本类实例。

调用父类的那个构造器执行初始化，分一下情况：
1.子类构造器执行体的第一行使用super显示调用父类构造器，系统将根据super调用传入的实参列表来确定调用父类哪个构造器
2.子类构造器执行体的第一行使用this显示调用本类中重载的构造器，系统将根据this调用传入的实参列表来确定调用本类的另一个构造器
3.子类构造器执行体中既没有super调用，也没有this调用，系统将会在执行子类构造器之前，，隐式调用父类无参构造器

final

final可以修改变量，被final修饰的变量再被赋初始值后，不能再被修改
final可以修饰方法，被final修饰的方法不能被重写
fianl可以修饰类，被final修饰的类不能被继承

实例变量

final修饰的变量必须显式指定初始值，且只能在三个位置指定：
① 定义final实例变量时指定
② 在非静态初始化块中为final实例变量指定初始值
③ 在构造器中为final实例变量指定初始值

执行“宏替换”的变量

对于一个final变量，不论是类变量、实例变量，还是局部变量，只要被final修饰并且在定义时指定了初始值，该初始值在编译时就被确定下来了，final变量本质已不再是一个变量而是相当于一个直接量

final修饰局部变量-javap分析对比图

final修饰符的一个重要用途就是定义“宏变量”。final变量在定义时就被指定初始值，而且初始化值可以在编译时就确定下来，那么这个final变量本质上就是一个“宏变量”，编译器会把程序中所有用到该变量的地方直接替换成该变量的值。
注意！！只有在定义final变量时就指定初始值，此变量才是宏变量。在外面定义final变量，但在构造器或者代码块中指定final变量的值，这种变量虽然被final修饰但并不是宏变量，不会进行“宏替换”

局部内部类（包含匿名内部类）中的局部变量默认有final修饰，不论写不写final,final修饰符都在。 在任何局部内部类中访问的局部变量默认也都有final修饰 强调局部内部类 是因为只有局部内部类（包含匿名内部类）才可以访问局部变量，普通静态内部类、非静态内部不能访问方法体内的局部变量。

为什么Java要求内部类访问的局部变量必须带final修饰符？

普通的局部变量作用域停留在该方法内，当方法执行结束后，该局部变量也随之消失；但内部类则可能产生隐式的“闭包（Closure）”，闭包将使得局部变量脱离他所在的方法继续存在。

由于内部类可能会扩大局部变量的作用域，如果被内部类访问的局部变量没有使用final修饰，说明该变量可以随意改动，会引起极大的混乱，所以java编译器要求局部变量必须带final修饰符。
